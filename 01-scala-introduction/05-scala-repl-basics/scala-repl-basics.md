## Scala REPL とは

REPL（レプル）とは、`Read-Eval-Print Loop` の略で、対話的にコードを実行できる環境のことである。

Scala には標準でこの REPL が用意されており、短いコードをその場で試したり、動作を確認したりできる。Javaでも、JDK 9 以降は `JShell` という REPL が用意されている。

## REPL を起動する

REPL を起動するには、ターミナルで次のコマンドを実行する。

```bash
scala
```

起動後は、次のように Scala の式をそのまま入力して実行できる。

```text
scala> 3 + 3
val res0: Int = 6
```

## `res0` とは何か

上の結果では、`3 + 3` の計算結果である `6` が `res0` という名前で保存されている。

そのため、次のように `res0` を使って続けて計算できる。

```text
scala> res0 + 30
val res1: Int = 36
```

ここでの `res0` や `res1` は、REPL が計算結果を自動的に保存するときに付ける名前である。前の結果をすぐ再利用できるので、試しながら学ぶときに便利である。

## `Int` は型を表している

結果の中に表示される `Int` は、この値の「型」を表している。

```text
val res0: Int = 6
```

これは、次の意味になる。

- `res0` という名前の値がある
- その型は `Int` である
- その値は `6` である

Scala では、すべてのデータが型の情報を持っている。`Int` は `Integer` の略で、整数を表す型である。今回の `6` は整数なので、型が `Int` と表示されている。

型は、データベースなどで学ぶデータ型と似た考え方であり、「その値がどのような種類のデータなのか」を表すための情報である。

## REPL を終了する

REPL を終了するには、次のコマンドを入力する。

```text
:quit
```

## 便利なコマンド

REPL では、いくつか便利なコマンドを使える。

- `:help`
  利用できるコマンドを確認する
- `:load <path>`
  指定したファイルを読み込んで実行する
- `:reset`
  これまでに定義した変数や関数などをリセットする

たとえば、ヘルプを確認するには次のように入力する。

```text
:help
```

## 最後

Scala の REPL を使うと、コードを 1 行ずつその場で実行しながら確認できる。文法の確認や、計算結果、変数、型の挙動を試すときに便利な学習用ツールである。

REPLはRead-Eval-Print Loopの略であり、対話実行環境のことを指す。ScalaのREPLでは、実行結果は自動でresとインデックスの数値で表される変数に代入される。また、REPLは:quitで終了でき、利用可能なコマンドは:helpで確認できる。更に、REPLはScalaだけでなく、他の多くのプログラミング言語（例えばPythonやJavaScriptなど）でも利用できるツール。
