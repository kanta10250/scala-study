## Scala で作成した Hello World を読む

ここでは、`01-run-scala-from-file` で作成した `Main.scala` の内容を読みながら、Scala の基本的な書き方を確認する。

対象となるソースコードは次のとおりである。

```scala
@main def hello = println("Hello World")
```

このプログラムは、実行すると `Hello World` と表示する最小構成の Scala アプリケーションである。

## `Main.scala` というファイル名

このプログラムは `Main.scala` という名前のファイルに書かれている。

- `Main`
  プログラムの入口となる処理をまとめたファイルとして、よく使われる名前である
- `.scala`
  このファイルが Scala で書かれていることを示す拡張子である

なお、`Main.scala` という名前は言語仕様で必須というわけではないが、最初のサンプルプログラムでは分かりやすい名前としてよく使われる。

## `@main` とは何か

ソースコードの先頭にある `@main` は、アノテーションである。

```scala
@main def hello = println("Hello World")
```

アノテーションは、プログラム中の定義に対して追加情報を与える仕組みである。ここでは、`hello` という関数を「プログラム実行時の入口として使う関数」であることを示している。

Scala では、従来の Java のように特別な形の `main` メソッドを直接書く代わりに、`@main` を使って実行可能な関数を定義できる。

## `def hello` とは何か

`def` は、関数を定義するときに使うキーワードである。

```scala
def hello = println("Hello World")
```

ここでは `hello` という名前の関数を定義している。

関数定義の基本形は次のようになる。

```scala
def 関数名 = 関数の処理
```

Scala 3 では、複数行の関数をインデントで書くこともできる。

```scala
def 関数名 =
  処理1
  処理2
```

また、ブレース `{}` を使って書くこともできる。

```scala
def 関数名 = {
  処理1
  処理2
}
```

この教材では、Scala 3 でよく使われるインデント中心の書き方を基本とする。

## `println("Hello World")` の意味

関数の本体になっているのは、次の部分である。

```scala
println("Hello World")
```

`println` は、引数として渡した内容をコンソールに表示し、その後に改行する関数である。

- `print`
  内容を表示する
- `ln`
  line の略で、改行を表す

そのため、`println("Hello World")` を実行すると、`Hello World` という文字列が表示され、その後に改行される。

Scala では、文字列は次のようにダブルクォーテーションで囲んで書く。

```scala
"Hello World"
```

## このプログラム全体の意味

ここまでをまとめると、このコードは次の流れになっている。

1. `hello` という関数を定義する
2. `@main` を付けて、その関数を実行時の入口にする
3. 関数の中で `println("Hello World")` を実行する
4. 結果として、画面に `Hello World` が表示される

## まとめ

このプログラムから分かる Scala の基本は、次のとおり。

- Scala のソースコードは `.scala` ファイルに書く
- `def` を使って関数を定義する
- `@main` を付けることで、実行可能な入口関数を作れる
- `println` を使うと文字列を表示して改行できる
- Scala 3 では、インデントを使った簡潔な書き方ができる(Pythonみたいな)

最小のプログラムであっても、Scala の関数定義、アノテーション、文字列、出力の考え方が含まれている。
