## sbt を起動したままプログラムを実行する

前の資料では、ターミナルで `sbt run` を実行して Scala のプログラムを動かした。ここでは、`sbt` を一度起動したままにして、その中で `run` などのコマンドを実行する方法を確認する。

## sbt を起動する

まず、Mac のターミナルや VS Code のターミナルで、プロジェクトのディレクトリに移動してから次のコマンドを実行する。

```bash
sbt
```

これで `sbt` が起動し、コマンドを受け付ける状態になる。

たとえば、次のようなプロンプトが表示される。

```text
sbt:project-name>
```

この状態では、`sbt` を毎回起動し直さなくても、続けてコマンドを実行できる。

## `run` を実行する

`sbt` が起動した状態で、次のように入力する。

```text
run
```

これにより、このプロジェクトにある `Main.scala` がビルドされ、実行される。

たとえば、正常に動作すると次のように表示される。

```text
Hello World
[success] Total time: 1 s, completed Apr 19, 2026, 3:20:19 PM
```

前の資料で使った `sbt run` は、「`sbt` を起動してから `run` を 1 回実行する」形であり、結果としては同じ意味になる。

## sbt を起動したまま使う利点

`sbt` を起動したままにしておくと、コードを修正した後に `run` を再実行するだけで変更内容を確認できる。

そのため、毎回 `sbt` を立ち上げ直すよりも、繰り返し実行するときに便利である。

## `lastGrep` を使ってログを確認する

`sbt` では、実行時のログを後から確認するためのコマンドも使える。

たとえば、次のように入力する。

```text
run
lastGrep debug
```

すると、`debug` という文字列を含むログだけを取り出して表示できる。

たとえば、次のような内容が表示される。

```text
[debug] > Exec(run, Some(fffda653-1c0d-4742-b5c3-9849c9fa7fc7), Some(CommandSource(console0)))
[debug] Evaluating tasks: Compile / run
[debug] Running task... Cancel: Signal, check cycles: false, forcegc: true
[debug] > Exec(shell, None, None)
```

通常、`sbt` のデバッグログはそのままでは細かく表示されないが、`last` や `lastGrep` を使うことで、必要な情報だけを後から確認できる。

## まとめ

この資料で確認した内容は、次のとおりである。

- `sbt` を単独で起動すると、常駐した状態でコマンドを受け付ける
- その状態で `run` を実行すると、Scala のプログラムをビルドして実行できる
- `sbt run` と `sbt` の中での `run` は、結果として同じプログラム実行につながる
- `lastGrep` を使うと、ログの中から必要な情報を検索できる

`sbt` を起動したまま使う方法を覚えておくと、プログラムの修正と実行を繰り返す作業を効率よく進められる。
