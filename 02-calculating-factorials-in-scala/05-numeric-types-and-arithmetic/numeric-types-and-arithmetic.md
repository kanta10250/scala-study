# Scala における数値型と算術演算

ここでは、Scala で数値をどのように表現し、どのように計算するのかを整理する。
前の章で扱った距離計算の例では、小数の計算結果に誤差が出た。ここでは、その理由も含めて数値型の基本を確認する。

## 数値は 2 進数で表現される

コンピューターの中のデータは、すべて `0` と `1` の組み合わせで表される。数値も同様である。

2 進数では、たとえば次のように表現する。

```text
1  = 1
2  = 10
3  = 11
4  = 100
```

Scala では、こうした内部表現の上に整数型や浮動小数点型が用意されている。

## 基本となる数値型

Scala でよく使う数値型は、まず `Int` と `Double` である。

- `Int` は整数を表す型
- `Double` は小数を含む数値を表す型

`Int` は 4 バイト、つまり 32 ビットで表現される。
扱える範囲は `-2147483648` から `2147483647` までである。

一方、`Double` は 8 バイト、つまり 64 ビットで表現される。
非常に大きい範囲の値を扱えるが、すべての小数を正確に表せるわけではない。

## 浮動小数点数で誤差が出る理由

`Double` は、64 ビットを次のように分けて使う。

- 1 ビット: 符号
- 11 ビット: 指数部
- 52 ビット: 仮数部

この形式では、小数を 2 進数ベースで表現するため、10 進数の小数を正確に表せない場合がある。

たとえば REPL で次を実行すると、

```scala
0.1 * 0.1
```

結果は次のようになる。

```text
val res0: Double = 0.010000000000000002
```

`0.01` ぴったりにならないのは、`Double` が近い値で近似して保持しているためである。
前の章の距離計算で小数に誤差が出たのも、同じ理由による。

## 主な数値型の一覧

Scala には、`Int` と `Double` 以外にも次のような数値型がある。

| 型 | 内容 | 記述例 |
| --- | --- | --- |
| `Byte` | 1 バイトの符号付き整数 | `1.toByte` |
| `Short` | 2 バイトの符号付き整数 | `1.toShort` |
| `Int` | 4 バイトの符号付き整数 | `1` |
| `Long` | 8 バイトの符号付き整数 | `1L` |
| `Float` | 4 バイトの浮動小数点数 | `1.0F` |
| `Double` | 8 バイトの浮動小数点数 | `1.0` |

REPL では、これらの値をそのまま入力して型を確認できる。

## 型変換

数値は、メソッドを使って別の型に変換できる。

```scala
1.toByte
1.toShort
1.toInt
1.toLong
1.toFloat
1.toDouble
```

こうした変換は、型の違いを確認したいときや、計算に使う型をそろえたいときに使う。

## 数値の書き方

Scala では、整数を 16 進数で書いたり、桁区切りにアンダースコアを使ったりできる。

```scala
0x0000FF
1_000_000_000
```

REPL で確認すると、次のようになる。

```text
scala> 0x0000FF
val res0: Int = 255

scala> 1_000_000_000
val res1: Int = 1000000000
```

## 数値以外の基本的な値型

Scala では、数値だけでなく文字や真偽値も基本的な値として扱う。

| 型 | 内容 | 記述例 |
| --- | --- | --- |
| `Char` | 2 バイトの Unicode 文字 | `'A'` |
| `String` | 文字列 | `"AB"` |
| `Boolean` | 真偽値 | `true`, `false` |

`Char` は 1 文字、`String` は文字の並び、`Boolean` は条件判定などで使う。

## 算術演算子

Scala では、基本的な算術演算子を使って計算できる。

| 計算 | 演算子 | 記述例 |
| --- | --- | --- |
| 足し算 | `+` | `1 + 1` |
| 引き算 | `-` | `10 - 7` |
| 掛け算 | `*` | `2 * 2` |
| 割り算 | `/` | `1 / 3` |
| 余り | `%` | `13 % 7` |

REPL で順番に試すと、数値型ごとの違いを確認しやすい。

## 整数の割り算に注意する

整数同士で割り算をすると、小数部分は切り捨てられる。

```scala
1 / 3
```

この結果は、次のようになる。

```text
val res0: Int = 0
```

小数を含む結果が必要なときは、少なくともどちらかを浮動小数点数にする。

```scala
1.0 / 3
```

すると、結果は `Double` になる。

```text
val res1: Double = 0.3333333333333333
```

## まとめ

- Scala の数値は、内部では `0` と `1` の 2 進数で表現される
- `Int` は整数、`Double` は小数を含む値を表す代表的な型である
- `Double` は便利だが、10 進数の小数を常に正確には表せない
- `toInt` や `toDouble` などのメソッドで型変換ができる
- 算術演算子は `+`、`-`、`*`、`/`、`%` が基本である
- 整数同士の割り算では小数部分が失われるため、必要に応じて `Double` を使う
