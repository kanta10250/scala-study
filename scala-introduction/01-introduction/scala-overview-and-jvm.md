## Scalaとは

2003年に2003年にスイス連邦工科大学のMartin Odersky教授(マーティン・オダースキー教授)によって開発されたプログラミング言語。

Scalaという名前自体が、Scalable Language（スケーラブルな言語）という意味を持っている。

Martin Odersky教授はJava 5で導入された機能であるジェネリクスの開発者であり、彼の書いたJavaコンパイラはJDK 1.3の頃から標準搭載されている。
JDKとは、Java Development Kitの略でJavaを使ってアプリケーションなどを作るソフトウェア開発キットおよび開発環境のことを指している。

## Javaについて

Javaプログラミング言語は、1995年に登場した言語で、30年以上の歴史の中で多くの実績やライブラリ資産を持っているプログラミング言語。

大規模のアプリでオブジェクト指向プログラミング言語の有用性がわかることで、オブジェクト指向言語であるJavaやC# などで実装されることが多くなっている。

Javaという言語は、JVM（Java Virtual Machine）という仮想マシン上で動くことを前提として設計されている。

本来、プログラムはCPUやOSなどの実行環境に合わせて作られる。しかしJavaは、ソースコードをそのままCPU向けの機械語に変換するのではなく、いったんJVMが理解できる「バイトコード」に変換して実行する。

JVM自体はさまざまなCPUやOS向けに用意されているため、対応するJVMさえあれば、同じJavaのプログラムを異なる環境でも動かせる。これは、Javaの特徴としてよく知られる「Write Once, Run Anywhere（一度書けば、どこでも動く）」という考え方につながっている。

Scalaも基本的にはこのJVM上で動作するため、Javaの豊富なライブラリ資産を活用しながら、さまざまな実行環境で動かせる。

Javaの仕組み。

```text
ソースコード（.java）
        |
        | コンパイル
        v
クラスファイル（.class / バイトコード）
        |
        | JVMが読み込んで実行
        v
プログラムが動く
```

### ScalaはJVMで動きJavaの資産を使える

Scalaは、オブジェクト指向プログラミングと関数型プログラミングの両方ができる。

またScalaは、オブジェクト指向や関数型プログラミングというものの他に、非同期プログラミング、並行・並列処理を得意としている。
